# AutoScript Hub 可用性完整修复设计

## 目标

修复当前中心服务端与 Windows 桌面客户端之间未闭合的安装、日志、结果文件、离线执行、连接检测、心跳看门狗和客户端设置链路，使项目能够从全新仓库完成安装，并在服务端与客户端位于不同电脑时正常工作。

## 已确认的产品边界

- 结果文件不上传服务器，始终保留在实际执行脚本的客户端电脑。
- 桌面端通过本地 Agent 打开结果文件；纯 Web 端只显示文件名、路径和执行机器提示，不提供下载按钮。
- 执行日志需要增量上报服务器，以支持 Web/桌面端实时查看和工单日志快照。
- 离线模式必须允许客户端在后端不可达时冷启动、浏览本地缓存脚本、执行脚本并查看本地执行历史和日志。
- 恢复联网后，离线执行记录和最终日志自动同步到服务器；结果文件仅同步元数据。

## 方案比较

### 方案 A：共享文件系统

让服务器直接读取客户端路径。实现改动最少，但要求所有电脑共享同一磁盘和路径映射，不符合当前“中心服务器 + 员工电脑”的部署方式，因此不采用。

### 方案 B：结果与日志全部上传

链路最统一，但会复制可能很大的结果文件，并与“结果文件暂不需要上传”的产品决定冲突，因此不采用。

### 方案 C：日志上传、结果本地保留（采用）

日志以带偏移量的增量请求上传服务器；结果文件仅上报结构化元数据。桌面端通过本地 Agent 打开本机结果，Web 端只展示位置说明。该方案满足远程可观测性，同时避免复制结果文件。

## 架构设计

### 安装与静态资源

安装器统一执行 `npm ci` 和 `npm run build`，再将 `frontend/dist` 同时复制到 `backend/static` 与 `client/ui/static`。缺少 Node/npm、依赖安装失败或构建失败都视为安装失败，不再继续输出“安装完成”。服务端与客户端安装都调用同一个构建函数，避免行为漂移。

### 在线日志

Agent 为每个在线 run 维护已上传字节偏移量，周期性读取新增日志并调用后端日志追加接口。后端将内容追加到服务器的标准 run 日志文件，并更新 run 的服务端日志路径。SSE 继续读取服务器日志，因此无需直接访问客户端文件系统。任务结束时 Agent 强制刷新剩余日志。

日志追加接口使用 run 所属用户身份和当前 Agent 身份进行关联。重复偏移或重试不会重复写入；服务器返回确认后的新偏移量。

### 本地结果文件

Agent 将 `main()` 返回的路径标准化为结果元数据列表，包含 `name`、`path`、`exists` 和可选的 `size`，写入 run 的 `result_files` 字段。桌面 UI 检测到本地 Agent 且 run 属于本机时，通过 `/local/results/open` 请求本地 Agent 打开路径。Web 页面只展示“文件保存在执行客户端”及元数据，不调用服务器 `os.startfile()` 或下载接口。

### 离线启动与页面路由

Agent 启动顺序调整为：加载本地缓存和待同步记录 → 启动本地 HTTP 服务 → 循环尝试认证和注册 → 在线轮询或离线维护。认证失败不再退出进程。

前端脚本列表、执行历史、执行详情和日志组件都根据连接状态选择后端 API 或 `/local/*` API。离线执行完成后进入执行历史页仍能看到本地 run；恢复联网并同步后显示后端 run。

### 连接检测

本地 Axios 实例保持稳定，`refresh` 回调不因渲染重新创建。连接检查在挂载时执行一次，之后固定每 30 秒执行；卸载时取消定时器。只有状态实际变化时更新连接状态。

### 心跳看门狗

所有从 SQLite 读出的时间在运算前统一转换为 UTC-aware datetime。看门狗测试覆盖失联 Agent、running run 和 duration 计算，确保异常不会回滚整次扫描。

### 客户端设置

后端只保存每用户设置，不再写服务器本地 `client_config.json`。Agent 在线时定期拉取当前用户设置，将允许的客户端字段合并到本机配置；桌面端保存设置后通知本地 Agent 立即刷新。需要重启才能生效的服务器地址明确提示用户重启客户端。

## 错误处理

- 日志上传失败进入待重试队列，不阻塞脚本执行。
- 日志偏移不一致时服务器返回当前偏移，Agent 从确认位置继续。
- 本地结果不存在时 UI 显示明确错误，不影响 run 的最终状态。
- 静态资源构建失败时安装器返回非零退出码。
- 后端离线期间 Agent 保持运行，采用有上限的退避重试，恢复后自动注册和同步。

## 测试与验收

- 安装器测试覆盖无 `node_modules` 的 fresh clone 路径，以及两处静态目录复制。
- 后端测试覆盖日志增量追加、偏移幂等、日志读取和心跳时区。
- 客户端测试覆盖离线冷启动、本地 run 列表/详情/日志、结果元数据和本地打开。
- 前端使用 Node 内置测试运行器，覆盖连接检测只按预定周期调用，以及离线脚本/执行历史路由。
- `pytest`、前端测试、`npm run lint`、`npm run build`、Python compileall 和安装/启动冒烟全部通过。
