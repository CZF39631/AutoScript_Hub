# AutoScript Hub 0.9 发布指南

## 版本规则

- `v0.9.0` 构建 `0.9.0` Beta；Docker 标签为 `0.9.0`、`0.9`、`beta`，不发布 `latest`。
- 通过全部验收后，从同一提交创建 `v1.0.0`；Docker 标签为 `1.0.0`、`1.0`、`stable`、`latest`。
- 客户端、服务端健康响应、安装器、更新清单和镜像版本必须一致。

## 发布前配置

GitHub Actions Secrets：

- `UPDATE_PRIVATE_KEY_B64`（必需）：Ed25519 私钥 PEM 的 Base64，仅存在于仓库 Secret。
- `GITEE_TOKEN`（可选）：公开 Gitee 镜像仓库的 Release 写入 Token。

启用 Gitee 镜像时再配置 GitHub Actions Variables：

- `GITEE_OWNER`
- `GITEE_REPO`

首次生成更新密钥：

```powershell
python release/scripts/generate_update_key.py `
  --private-key .release-secrets/update-private-key.pem `
  --public-key client/update/update-public-key.b64
```

将私钥 PEM 整体做 Base64 后保存为 `UPDATE_PRIVATE_KEY_B64`。私钥、真实 Token、`deploy/.env` 和生成资产不得提交 Git；客户端只携带公钥。

## 本地 Windows 构建

需要 Python 3.11、Node.js 20 和 Inno Setup 6：

```powershell
release/windows/build.ps1 -Version 0.9.0 -PythonExe python
```

脚本会运行 Python/前端测试，构建 `AutoScriptHub.exe`、`AutoScriptAgent.exe`、`AutoScriptUpdater.exe`，从指定的 Python 3.11.9 构建环境生成私有便携运行时、下载并校验 WebView2 引导程序，最后生成安装器并执行 95 MiB 门槛。

## 标签发布

```bash
git tag v0.9.0 <已验收提交SHA>
git push origin v0.9.0
```

`.github/workflows/release.yml` 将：

1. 重跑 CI。
2. 生成 Windows 安装包。
3. 构建并推送 `linux/amd64`、`linux/arm64` 多架构 GHCR 镜像，并分别启动健康冒烟。
4. 生成部署包和 Skill ZIP。
5. 创建并发布 GitHub Release。
6. 生成 Ed25519 签名更新清单并上传，重算所有最终资产的 `SHA256SUMS.txt`。
7. 如果 `GITEE_TOKEN`、`GITEE_OWNER`、`GITEE_REPO` 三项都已配置，再同步并匿名校验 Gitee Release；未配置时不阻塞 GitHub 发布。

## 必需资产

- `AutoScript-Hub-Setup-<version>.exe`
- `autoscript-hub-update.json`
- `autoscript-hub-update.json.sig`
- `SHA256SUMS.txt`
- `autoscript-script-authoring-<version>.zip`
- `autoscript-hub-server-deploy-<version>.zip`
- GHCR 多架构镜像 `linux/arm64` 与 `linux/amd64`

校验：

```bash
sha256sum -c SHA256SUMS.txt
```

GitHub 是必需的发布真源；启用 Gitee 时，其 URL 排在 GitHub URL 前，两个站点必须匿名下载同一安装包、清单和签名。客户端先验签清单，再校验安装包长度和 SHA-256。

## 0.9 到 1.0

只有 `docs/0.9-acceptance-checklist.md` 的“0.9 简明交付门槛”全部有证据、用户明确确认后才能晋级；后续明细仍用于声明具体兼容范围：

1. 记录通过验收的精确提交 SHA。
2. 冻结该提交，不再混入新改动。
3. 在同一提交创建 `v1.0.0`。
4. 由 CI 重新构建 Stable 产物，不重命名 0.9 二进制。
5. 对 1.0 安装包、更新、回滚和双架构镜像再次冒烟。
